<p>
 Aspose.Words untuk Android melalui Java adalah API pemrosesan dokumen yang memungkinkan Anda melakukan tugas manipulasi dokumen secara langsung dalam aplikasi Android Anda. API memungkinkan Anda membuat, memanipulasi, mengonversi, dan merender file Microsoft Word dengan mudah.
</p>

<p>
 Aspose.Words untuk Android melalui Java mendukung pemrosesan format asli seperti DOC, DOCX, OOXML, TXT, RTF serta banyak format lain yang umum digunakan seperti HTML, OpenDocument ODT, PDF, EPUB, XPS, dan format gambar seperti JPEG, PNG , SVG dan EMF.
</p>

    <h2 class="h2title">
     Konversi Format File
    </h2>
    <p>
     Kemampuan untuk mengonversi berbagai file dengan cepat dan andal dengan tingkat presisi yang tinggi adalah alasan sebagian besar pelanggan kami memilih API. Android Microsoft Word API memungkinkan Anda mengonversi file dari format apa pun yang didukung ke format lain yang didukung, mis., mengonversi dokumen apa pun ke SWF dengan fidelitas tinggi, dan SWF yang dihasilkan berisi halaman dokumen dan penampil tersemat yang memungkinkan dokumen dilihat di perangkat apa pun. Pengembang dapat dengan mudah mengonversi format hanya menggunakan dua baris kode.
    </p>
    <div class="codeblock" id="code">
     <h3>
      Simpan dokumen dalam format yang berbeda - Java
     </h3>
     <pre><code class="java">// memuat file yang akan dikonversi

Document doc = new Document(SD_PATH + "template.doc");

// Konversi DOC ke DOCX, PDF, dan HTML

doc.save(SD_PATH + "output.docx", SaveFormat.DOCX);

doc.save(SD_PATH + "output.pdf", SaveFormat.PDF);

doc.save(SD_PATH + "output.html", SaveFormat.HTML);</code></pre>
    </div>

    </h2>
    <p>
     Library ini memberikan dukungan ekstensif untuk memuat, melihat, menyimpan, dan mengonversi file WordprocessingML. Implementasi komprehensif semacam itu dimungkinkan karena API dirancang dengan mempertimbangkan struktur dokumen Microsoft Word (dan WordprocessingML diketahui meniru representasi internal file Microsoft Word) sehingga memudahkan kami untuk menyertakan dukungan untuk implementasi ini ke mesin produk inti kami.
    </p>
   </div>
   <div class="col-lg-12">
    <h2 class="h2title">
     Muat dan Simpan Format HTML
    </h2>
    <p>
     Impor HTML dan XHTML Android Word API tahan terhadap dokumen yang tidak dibentuk dengan baik. Elemen atau atribut yang tidak valid atau tidak dikenal akan diabaikan dengan aman selama proses impor sedangkan konversi ke format HTML dan XHTML dapat disesuaikan. Banyak parameter seperti ekspor properti dokumen, header dan footer, lokasi gambar, penskalaan dan resolusi, opsi ekspor bidang formulir, dan lainnya, semuanya dapat ditentukan.
    </p>
   </div>
   <div class="col-lg-12">
    <h2 class="h2title">
     Manipulasi dengan Model Objek Dokumen
    </h2>
    <p>
     Pustaka dokumen Android memungkinkan pengembang untuk membuat, membangun, dan memodifikasi file - termasuk pemformatan dengan model objek dokumen intuitif yang terdiri dari lebih dari 100 kelas.
    </p>
    <p>
     Pengembang Android secara terprogram dapat membuat, memodifikasi, mengekstrak, dan mengganti semua elemen termasuk bagian, header, footer, paragraf, daftar, tabel, teks, bidang, hyperlink, bookmark, dan gambar. API juga memungkinkan penentuan pemformatan terperinci untuk objek gambar termasuk bentuk, kotak teks, gambar, objek OLE, dan kontrol.
    </p>
   </div>
   <div class="col-lg-12">

     Pelaporan dan Gabungan Surat

    <p>

    </p>
    <p>
     API memungkinkan pengembang untuk menggunakan bidang gabungan surat Microsoft Word standar dalam laporan. Selain itu, Anda dapat menggunakan sintaks API yang diperluas untuk bidang gabungan surat untuk melakukan operasi yang lebih kompleks selama operasi gabungan surat seperti menyisipkan gambar atau wilayah yang dapat diulang. Data yang mendorong laporan dan gabungan surat Anda dapat berasal dari objek ResultSet, objek bisnis, atau dari sumber data kustom sepenuhnya.
    </p>
